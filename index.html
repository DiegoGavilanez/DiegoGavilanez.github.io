Proyecto destacado

Enterprise Integration Service

Este caso demuestra cómo abordo problemas reales de comunicación entre sistemas. No publico código fuente porque pertenece al entorno empresarial donde fue construido; presento el problema, la arquitectura, las decisiones técnicas y los resultados.

Problema Procesos distribuidos entre WinDev, Visual FoxPro y bases de datos necesitaban comunicarse de forma más ordenada.
Diseño Servicio intermedio desacoplado con endpoints HTTP, comunicación JSON y ejecuciones batch, on demand y background.
Resultado Mayor centralización, integración más clara y una base lista para seguir evolucionando hacia tecnologías modernas.

Arquitectura demostrada

  • Windows Service
  • HTTP Endpoints
  • Comunicación JSON
  • Detección de puertos disponibles
  • Arquitectura desacoplada
